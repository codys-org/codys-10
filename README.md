# Defensible 10 Standards

[![License: CC BY 4.0](https://img.shields.io/badge/License-CC%20BY%204.0-lightgrey.svg)](https://creativecommons.org/licenses/by/4.0/)

The **Defensible 10** is a set of 10 essential cybersecurity standards designed to provide organizations with a clear, actionable framework for building a defensible security posture. This repository hosts the official standards, documentation, and resources.

Visit the project website: [https://www.defensible10.org](https://www.defensible10.org)

---

## The 10 Standards

1. **Inventory & Control of Hardware Assets**  
   Actively manage all hardware devices on the network.

2. **Inventory & Control of Software Assets**  
   Actively manage all software on the network.

3. **Continuous Vulnerability Management**  
   Continuously acquire, assess, and take action on new information.

4. **Controlled Use of Administrative Privileges**  
   Track, control, prevent, and correct the use of admin rights.

5. **Secure Configuration for Hardware and Software**  
   Establish, implement, and actively manage secure configurations.

6. **Maintenance, Monitoring, and Analysis of Audit Logs**  
   Collect, manage, and analyze audit logs of events.

7. **Email and Web Browser Protections**  
   Minimize the attack surface and opportunities for attackers.

8. **Malware Defenses**  
   Control the installation, spread, and execution of malicious code.

9. **Limitation and Control of Network Ports, Protocols, and Services**  
   Manage network-based services and access.

10. **Data Recovery Capabilities**  
    Ensure the ability to restore data and systems after an incident.

## Contributing

We welcome contributions from the cybersecurity community! Whether you're improving documentation, adding translations, or suggesting enhancements to the standards, your input is valued.

### How to Contribute

1. **Fork** the repository.
2. **Create** a new branch (`git checkout -b feature/your-feature`).
3. **Commit** your changes (`git commit -m 'Add your feature'`).
4. **Push** to the branch (`git push origin feature/your-feature`).
5. **Open** a Pull Request.

Please ensure:
- Your changes follow the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md).
- All Markdown files are properly formatted.
- You reference the standard number and version in your PR.

### Suggesting Changes to Standards

Propose changes via GitHub Issues using the **"Standard Enhancement"** template. Include:
- Standard number
- Current text
- Proposed change
- Rationale and supporting evidence
